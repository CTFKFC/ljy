# 烬与月｜越集（重构维护版）

我爱你

希望你永远看不到这个，不然我的印象分就会又要降低了，虽然你已经把我删掉了

自己写的网站里面的文章，网站代码包括这个README.md都是ai写的

- 首页 `index.html` 只负责气氛、目录、摘读入口
- `content.js` 只保留元数据：标题、slug、摘要、精选标记
- **完整正文只保留在 `poems/*.html`**
- `script.js` 只根据元数据生成首页目录和摘读卡片，不再把全文塞进首页

## 结构
- `index.html`：首页
- `content.js`：目录元数据（轻量）
- `script.js`：生成首页目录和摘读入口
- `poems/*.html`：每篇文章的唯一正文来源
- `styles.css`：全站样式

## 以后新增文章怎么做
1. 在 `poems/` 里新增一个单篇 html，文件名保持这种格式：
   - `34-新文章标题.html`
2. 打开 `content.js`
3. 在 `poems` 数组末尾新增一项：

```js
{
  "title": "新文章标题",
  "slug": "34-新文章标题",
  "excerpt": "这里写首页显示的简短摘要"
}
```

4. 如果想让它进入首页“摘读”，把标题加入 `featured_titles`
5. 保存并重新上传部署

## 以后修改文章怎么做
- **正文内容**：只改 `poems/对应文件.html`
- **首页标题/摘要/排序入口**：改 `content.js`

这样不会再出现“首页一份正文、单篇页又一份正文”的重复维护问题。

## 以后删除文章怎么做
1. 从 `content.js` 里删掉对应项
2. 删除 `poems/` 里对应 html
3. 如果它在 `featured_titles` 中，也一起删掉

## 现在这版的原则
- 首页轻
- 单篇页承载正文
- 后续加文章不会越来越卡
- 内容维护只有一处正文来源
